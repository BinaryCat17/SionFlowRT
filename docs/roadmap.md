# Roadmap проекта SionFlowRT (Обновленный)

## Этап 1: Базовая функциональность (Выполнено ✅)
- [x] Инициализация проекта на Rust.
- [x] Описание структуры графа в JSON.
- [x] Топологическая сортировка и валидация DAG.
- [x] Базовые операции (Add, Mul, Sin).
- [x] Механизм страйдов и Broadcasting.
- [x] Автоматический Loop Fusion.
- [x] Генерация C-кода с OpenMP.

## Этап 2: Расширение вычислительных возможностей (Выполнено ✅)
- [x] **Константы:** Поддержка узлов-констант, встроенных прямо в код.
- [x] **Транспонирование:** Реализация через манипуляцию страйдами.
- [x] **Редукция:** Сумма по осям (Sum).
- [x] **Матричные операции:** MatMul (умножение матриц).
- [x] **Универсальная свёртка:** N-D Convolution.

## Этап 3: Манифест и Системные Ресурсы (Выполнено ✅)
- [x] **Window Management via Manifest:** Декларативное создание системного окна SDL2 через описание в манифесте.
- [x] **Interactive Device Mapping:** Привязка мыши (координаты) к входным тензорам графа.
- [x] **Display Sinks:** Маппинг выходных тензоров на экран через универсальный механизм Mappings.
- [x] **Dynamic Shape Propagation:** Автоматическая передача размеров окна в размеры тензоров через символьные константы в C.
- [x] **Stateful Orchestration:** Соединение нескольких программ и реализация Feedback loops (обратной связи) через двойную буферизацию в рантайме.

## Этап 3.5: Абстракция Рантаймов и Параметризация (Выполнено ✅)
- [x] **Agnostic Codegen:** Очистка генератора кода от системно-специфичных имен (WINDOW_WIDTH, SDL_...).
- [x] **Context Parameters:** Поддержка именованных параметров в манифесте, пробрасываемых в C как внешние константы.
- [x] **Pluggable Runtimes:** Разделение на `sdl2_interactive` и `headless_batch` оболочки.
- [x] **Unified Linker:** Выбор и линковка нужного рантайма на этапе сборки.

## Этап 4: Продвинутая модульность и системная интеграция (Выполнено ✅)
- [x] **Directory Imports:** Импорт папок и использование графов как библиотечных модулей.
- [x] **Deep Inlining:** Рекурсивное встраивание всех графов, ресурсов и драйверов в монолитный исполняемый код.
- [ ] **Advanced Symbolic Shape Propagation:** Интеллектуальный вывод размерностей тензоров на основе графовых зависимостей и символьных констант.
- [ ] **Standalone App Generation:** Сборка проекта в одно автономное приложение, готовое к запуску в ОС.

## Этап 5: Оптимизация памяти и потоков
- [ ] **Static Memory Mapping:** Полное статическое распределение всех буферов системы на этапе компиляции.
- [ ] **Cross-System Memory Reuse:** Оптимизация RAM между независимыми подсистемами.
- [ ] **Zero-Allocation Execution:** Гарантированное отсутствие вызовов malloc/free во время работы приложения.
