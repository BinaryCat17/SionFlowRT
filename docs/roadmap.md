# Roadmap проекта SionFlowRT (Обновленный)

## Этап 1: Базовая функциональность (Выполнено ✅)
- [x] Инициализация проекта на Rust.
- [x] Описание структуры графа в JSON.
- [x] Топологическая сортировка и валидация DAG.
- [x] Базовые операции (Add, Mul, Sin).
- [x] Механизм страйдов и Broadcasting.
- [x] Автоматический Loop Fusion.
- [x] Генерация C-кода с OpenMP.

## Этап 2: Расширение вычислительных возможностей (Выполнено ✅)
- [x] **Константы:** Поддержка узлов-констант, встроенных прямо в код.
- [x] **Транспонирование:** Реализация через манипуляцию страйдами.
- [x] **Редукция:** Сумма по осям (Sum).
- [x] **Матричные операции:** MatMul (умножение матриц).
- [x] **Универсальная свёртка:** N-D Convolution.

## Этап 3: Манифест и Системные Ресурсы (В процессе ⏳)
- [ ] **Window Management via Manifest:** Декларативное создание системного окна через описание в манифесте.
- [ ] **Interactive Device Mapping:** Привязка мыши (координаты, кнопки) и клавиатуры к входным тензорам графа.
- [ ] **Display Sinks:** Маппинг выходных тензоров на экран или текстуры окна.
- [ ] **Dynamic Shape Propagation:** Автоматическая передача динамических параметров (например, разрешения окна) в размеры тензоров.
- [ ] **Stateful Orchestration:** Соединение систем и зацикливание через Feedback loops с двойной буферизацией.

## Этап 4: Продвинутая модульность и системная интеграция
- [ ] **Directory Imports:** Импорт папок и использование графов как библиотечных модулей.
- [ ] **Deep Inlining:** Рекурсивное встраивание всех графов, ресурсов и драйверов в монолитный исполняемый код.
- [ ] **Standalone App Generation:** Сборка проекта в одно автономное приложение, готовое к запуску в ОС.

## Этап 5: Оптимизация памяти и потоков
- [ ] **Static Memory Mapping:** Полное статическое распределение всех буферов системы на этапе компиляции.
- [ ] **Cross-System Memory Reuse:** Оптимизация RAM между независимыми подсистемами.
- [ ] **Zero-Allocation Execution:** Гарантированное отсутствие вызовов malloc/free во время работы приложения.
